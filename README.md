# BlockerPasses-CS2
Блокирует проходы, если на сервере недостаточно игроков

## Возможности
- **Интеграция с MenuManager**: Использует MenuManagerCS2 для улучшенной системы меню
- **Резервная поддержка**: Возвращается к нативному ChatMenu, если MenuManager недоступен
- **Меню управления**: Простой в использовании интерфейс меню для администраторов
- **Инструменты позиционирования**: Получить текущую позицию и углы обзора
- **Информация о сущностях**: Просмотр подробной информации о сущностях карты
- **Управление конфигурацией**: Перезагрузка конфигурации и просмотр настроек
- **🎨 Система текстур**: Создание и применение пользовательских текстур к блокам
- **🖼️ Поддержка изображений**: Поддержка пользовательских текстур изображений и паттернов
- **📋 Готовые текстуры**: Готовые к использованию текстуры, включая 2x2 паттерн
- **🌐 Многоязычность**: Поддержка английского, русского и украинского языков
- **✏️ Редактирование блоков**: Изменение позиций и углов блоков через меню
- **📝 Логирование в файл**: Все логи плагина записываются в файл `logs/plugin_log.txt` вместо консоли сервера

## Установка
1. Установите [CounterStrike Sharp](https://github.com/roflmuffin/CounterStrikeSharp), [Metamod:Source](https://www.sourcemm.net/downloads.php/?branch=master) и [ResourcePrecacher](https://github.com/Pisex/ResourcePrecacher/releases/tag/1.0f)
2. **Опционально**: Установите [MenuManagerCS2](https://github.com/MenusMM/MenuManagerCS2) для улучшенной системы меню
   - Скачайте и соберите MenuManagerCS2
   - Скопируйте `MenuManagerApi.dll` в папку `3rd_party/`
3. Клонируйте этот репозиторий и соберите проект
4. Поместите `BlockerPasses.dll` в директорию плагинов

### После установки ResourcePrecacher, все пути, которые вы пишете в конфиге, также пишите там

## Команды

### Основные команды
`css_bp_reload`, `!bp_reload` - перезагружает конфигурацию (только для `@css/root`)
`css_bp_getpos`, `css_bp_geteye` - получает текущую позицию и углы обзора (только для `@css/root`)
`css_bp_menu`, `css_bp` - открывает меню управления (только для `@css/root`)

### Команды управления текстурами
`css_bp_createtexture <имя> <отображаемое_имя> [путь_к_текстуре] [категория]` - создает новую текстуру
`css_bp_applytexture <индекс_блока> <имя_текстуры>` - применяет текстуру к блоку
`css_bp_textures` - показывает список всех доступных текстур

### Команды управления блоками
`css_bp_add [невидимость] [лимит]` - добавляет блок на текущей позиции
`css_bp_list` - показывает список всех блоков на текущей карте
`css_bp_removeall` - удаляет все блоки с текущей карты

### Команды редактирования блоков
`css_bp_setorigin <индекс_блока> <x> <y> <z>` - устанавливает новую позицию для блока
`css_bp_setangles <индекс_блока> <x> <y> <z>` - устанавливает новые углы для блока

## Система текстур

### Создание текстур
```bash
# Создать пользовательскую текстуру
css_bp_createtexture my_logo "Логотип сервера" materials/logos/server_logo.vmt branding

# Создать текстуру с 2x2 паттерном
css_bp_createtexture 2x2_pattern "2x2 Паттерн" null patterns
```

### Применение текстур
```bash
# Применить текстуру к блоку
css_bp_applytexture 1 my_logo
css_bp_applytexture 2 2x2_pattern
```

### Готовые текстуры
- **white_block** - Классический белый блок
- **blue_block** - Синий цветной блок
- **red_block** - Красный цветной блок
- **green_block** - Зеленый цветной блок
- **2x2_pattern** - 2x2 шахматный паттерн (как в соревновательном режиме)

## Система меню

Этот плагин поддерживает как **MenuManagerCS2**, так и **нативную систему ChatMenu CounterStrikeSharp**, обеспечивая гибкость и расширенную функциональность.

### Интеграция с MenuManagerCS2
- **Улучшенный UI**: Лучший визуальный дизайн и пользовательский опыт
- **Расширенные возможности**: Больше опций меню и кастомизации
- **Единый интерфейс**: Объединенная система меню между плагинами
- **Автоматическое обнаружение**: Автоматически использует MenuManager, если доступен

### Резервная поддержка
- **Нативный ChatMenu**: Возвращается к встроенной системе меню CounterStrikeSharp
- **Без внешних зависимостей**: Работает "из коробки" без дополнительных плагинов
- **Надежная производительность**: Стабильная система меню с правильной обработкой ошибок
- **Простая разработка**: Простой API для создания и управления меню

### Опции меню
1. **🔄 Перезагрузить конфиг** - Перезагружает конфигурацию плагина
2. **📍 Получить позицию** - Получает текущую позицию и углы игрока (если включено)
3. **👁️ Получить углы обзора** - Получает текущие углы обзора игрока (если включено)
4. **⚙️ Текущие настройки** - Показывает текущий минимум игроков и название карты
5. **🗺️ Сущности карты** - Показывает все сущности, настроенные для текущей карты
6. **🎨 Управление текстурами** - Управление текстурами и их применение к блокам
7. **✏️ Редактировать позиции блоков** - Редактирование позиций и углов блоков

## Логирование
Все сообщения плагина, которые ранее выводились в консоль сервера, теперь записываются в файл `logs/plugin_log.txt`. Это включает:
- Информацию о загрузке конфигурации
- Сообщения о создании и применении текстур
- Логи позиционирования и редактирования блоков
- Другие отладочные сообщения

Файл логов автоматически создается в папке `logs/` относительно директории плагина.

## Требования
- CounterStrikeSharp с runtime
- **Опционально**: MenuManagerCS2 для улучшенной системы меню
  - Требуется `MenuManagerApi.dll` в папке `3rd_party/`
  - Требуются плагины PlayerSettings и AnyBaseLibCS2
